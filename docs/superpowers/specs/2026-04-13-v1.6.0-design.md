**Date:** 2026-04-13  
**Scope:** Tendencias de Laboratorio + Rename Expediente + Fix Auto-update Naming

## 1. Simple Fixes

### 1a. Rename Tab: "Nota de Evolución" → "Expediente"

**What:** The top-level tab label at `public/index.html` line ~438–440 reads "Nota de Evolución". Change visible label to "Expediente".

**Scope:** HTML label text only. Internal JS IDs (`apptab-nota`, `appcontent-nota`, `switchAppTab('nota')`) remain unchanged — purely cosmetic rename.

**Inner tabs** inside Expediente remain: "Nota de Evolución", "Indicaciones", + new "Tendencias" (see Section 3).

---

### 1b. Fix Auto-update Artifact Naming (permanent)

**Problem:** `productName: "R+"` causes electron-builder to sanitize `+` using the package `name` (e.g. `r-plus`) in `latest-mac.yml` URLs. Built files must match what the manifest lists to avoid manual asset renaming on release.

**Fix:** Change `artifactName` in `package.json` to a name with no special characters:

```json
"artifactName": "R-plus-${version}-${arch}.${ext}"
```

Both built files and manifest URLs will consistently use `R-plus-` prefix. No more manual renaming.

**Applies to:** `"build"` section in `package.json`. Both mac and win targets inherit this.

## 2. Lab History Data Model

### Current State
Labs stored as newline-delimited text in `notes[patientId].estudios`. Only 2 time slots exist: "anterior" (lines 0–2) and "reciente" (lines 3+). No structured numeric history.

### New Model

**localStorage key:** `rpc-labHistory`  
**Shape:** `{ [patientId: string]: LabSet[] }` — ordered oldest → newest.

```js
// LabSet shape
{
  id: string,       // Date.now().toString()
  fecha: string,    // "DD/MM/YYYY" from parsed lab report header
  hora: string,     // "HH:MM" from parsed lab report header
  resLabs: string[], // formatted lab strings (same as current, unchanged)
  parsed: {
    // BH
    Hb: number|null, Hto: number|null, Leuco: number|null, Plaq: number|null,
    // QS
    Glu: number|null, Cr: number|null, BUN: number|null, PCR: number|null,
    AU: number|null, TGL: number|null, COL: number|null,
    // ESC / Gaso
    Na: number|null, K: number|null, Cl: number|null, HCO3: number|null, Ca: number|null,
    // PFH
    AST: number|null, ALT: number|null, FA: number|null, BT: number|null,
  }
}
```

Null values mean that parameter was absent in the report.

### Migration (one-time, non-destructive)

On app load, after `patients` and `notes` are parsed from localStorage:

```
if rpc-labHistory does not exist:
  for each patient that has notes[id].estudios with content:
    parse the "reciente" slot → create LabSet with fecha from notes[id].fecha
    if "anterior" slot also has content → create second LabSet inserted before the reciente one, with fecha labeled literally as "Anterior" (no real date available)
  save rpc-labHistory
```

Migration is skipped on subsequent loads. Original `notes[id].estudios` is untouched.

### Write Path (on "Enviar a nota")

Existing flow (writing to `estudios`) is **unchanged**. After it completes, additionally:
1. Extract numeric values from `result.parsed` (already available from `parseLab_()`)
2. Extract numeric values by running a new `extractParsedValues(resLabs)` helper that re-parses the formatted strings (e.g. `"Hb 9.2*"` → `{ Hb: 9.2 }`) using regex — same patterns already used by `parsearSecciones()`
3. Build a new `LabSet` with the extracted values
4. Push to `labHistory[patientId]`
5. Save `rpc-labHistory` to localStorage

No UI change to the existing Laboratorio tab.

## 3. Tendencias de Laboratorio — UI

### Location

New inner tab `Tendencias` inside the `Expediente` tab, added after `Indicaciones`:

```
Expediente tab:
  [Nota de Evolución]  [Indicaciones]  [Tendencias]
```

### Chart.js

- Download `chart.umd.min.js` from Chart.js v4 releases
- Bundle at `public/vendor/chart.umd.min.js` (~210kb)
- Load via `<script src="vendor/chart.umd.min.js">` in `index.html` head
- No CDN dependency

### Sparkline Grid View (default)

Rendered when `Tendencias` tab is active for the selected patient.

- **Empty state:** If `labHistory[patientId]` has 0 or 1 entries → show: *"Agrega al menos 2 sets de laboratorio para ver tendencias"*
- **Grid:** CSS grid, 3 columns on wide layout, 2 on narrow
- **Sparkline card** (one per parameter that has ≥2 non-null values across history):
  - Header: parameter name (e.g., "Hb") + unit (e.g., "g/dL")
  - Most recent value — large, bold. Red if flagged with `*`, default color if normal
  - Mini Chart.js `line` chart, 80px tall, no axes labels, no legend
  - X: dates; Y: values; line color: `var(--action)` (blue)

Parameters shown (if data available): `Hb, Hto, Leuco, Plaq, Glu, Cr, BUN, PCR, Na, K, Cl, HCO3, Ca, AST, ALT, FA, BT, TGL, COL`

### Detail View (click sparkline card)

- Full-screen modal overlay (same backdrop pattern as SOAP modal)
- Large Chart.js `line` chart, full width, ~350px tall
- X axis: date labels formatted "DD/MM"
- Y axis: numeric values with unit
- Reference range shaded band (gray, 10% opacity) if min/max known
- All data points shown; hover tooltip: date + exact value
- "Cerrar" button top-right + click-outside-to-close
- Only one chart shown at a time

### Re-render trigger

Grid and charts re-render whenever:
- `Tendencias` inner tab is activated
- A new lab set is added to the patient (after "Enviar a nota")
